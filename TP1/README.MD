# TP1: Conceptos básicos para la construcción de Sistemas Distribuidos

## Practica
Un sistema distribuido surge como respuesta a la necesidad de escalar horizontalmente. Es fundamental comprender que en un sistema distribuido, se requiere la presencia de al menos dos nodos interconectados.
En el corazón de un sistema distribuido yace el procesamiento asincrónico y descentralizado.
La esencia del procesamiento asincrónico entre dos nodos radica en la comunicación entre ellos, donde el protocolo utilizado juega un papel central.
Para comprender este concepto de manera más concreta, imaginemos dos procesos, A y B. A desea iniciar una interacción con B y espera una confirmación de recepción.
Toda comunicación en red se basa en el principio de cliente-servidor. En este contexto, consideramos al nodo que está pasivamente a la espera de una solicitud como el servidor, que en este caso sería B, y al nodo que toma la iniciativa para iniciar la comunicación como el cliente, que en este caso sería A.
